# Калькуляция в таблице

## **Принцип  работы модуля «Табличная калькуляция».**

В отличие от рецептной калькуляции по деталям, табличная калькуляция предполагает добавление деталей и модулей только для формирования характеристик товара и отображения параметров для выбора на сайте, без учета цен из Справочника. \
Характеристики (параметры) генерируются системой во все возможные комбинации, вам достаточно заполнить в специальной форме – табличке цены за шт. или тираж  на сгенерированные сочетания параметров. \
При выборе клиентом на сайте параметров изделия, система будет автоматически подставлять внесенную в табличку цену.

Табличная калькуляция отменяет калькуляцию из Справочника, поэтому параметры не участвующие в  выборе клиентом, можно не включать в калькуляцию продукта (например, резка).&#x20;

{% hint style="info" %}
**Важно!**

**Ограничение использования:** Табличная калькуляция применяется только при выборе Стандартных и /или Индивидуальных размеров. Если в продукте используется Меняющиеся размеры табличная калькуляция работать не будет.&#x20;

**Особенности использования:** Табличная калькуляция не будет работать, если не заполнены размеры в основном модуле (заполнены только в деталях).&#x20;
{% endhint %}



## Заполнение цен при помощи табличной калькуляции

После заполнения Основных модулей, деталей и модулей из Справочника для быстрого заполнения продукции воспользуйтесь кнопкой в правом верхнем углу  «Калькуляция в таблице».   &#x20;

![](<../../../../.gitbook/assets/image (2256).png>)

\
Табличная калькуляция может применяться как при указании количества в тиражах (<mark style="background-color:yellow;">Модуль "Тиражность" → Выбор вариантов → Выбор из списка</mark>), так и при ручном вводе клиентом (<mark style="background-color:yellow;">Модуль "Тиражность" → Выбор вариантов → Ручной ввод</mark>). \
В открывшейся форме бегунками вы можете выбрать те операции/модули которые будут включаться в цену.\
В случае, если модуль не выбран, к табличной цене будет добавляться расчет цены по этому модулю из Справочника.

{% hint style="info" %}
**Важно!** Перед генерацией таблицы, обязательно должно быть включено <mark style="background-color:yellow;">отображение размеров</mark> в рецептной калькуляции (переключатель "Показывать на сайте"), иначе табличные цены не будут применяться в калькуляции на сайте. После генерации таблицы, размеры можно будет отключить, если это требуется.
{% endhint %}

![](<../../../../.gitbook/assets/image (3058).png>)

После сохранения списка выбранных модулей, откроется форма для заполнения, в которой слева генерируются сочетания всех параметров, а справа поля для заполнения цены за тираж или за ед. продукции.&#x20;

### **Группы клиентов**

Если у вас подключен модуль <mark style="background-color:yellow;">"Группы клиентов"</mark>,  в таблице так же будут учтены поля для заполнения для всех групп клиентов.

<figure><img src="../../../../.gitbook/assets/image (3859).png" alt=""><figcaption></figcaption></figure>

### **Себестоимость**

Если у вас подключен модуль <mark style="background-color:yellow;">"Себестоимость"</mark>, то вы можете указать себестоимость продукции. Он необходим для формирования отчёта по выручке, себестоимости и прибыли.

## **Пример заполнения таблиц**

### **Таблица с фиксированным выбором тиражей**

**Пример таблицы** для заполнения при **выборе тиражей** (<mark style="background-color:yellow;">Калькуляция → Модуль Тиражность →Выбор из списка</mark>):

<figure><img src="../../../../.gitbook/assets/image (3858).png" alt=""><figcaption></figcaption></figure>

В случае если количество выбрано по тиражам (как на изображении выше), для удобства заполнения предусмотрены следующие  возможности:

1\.  скачать (кнопка "Скачать" в верхнем правом углу)  для заполнения готовую таблицу с параметрами  в формате exel . После заполнения цен в таблице, загрузите (кнопка "Загрузить") ее обратно и цены распределятся по тиражам. Чтобы удалить табличную калькуляцию нажмите "Удалить все".

{% hint style="info" %}
**В скаченном файле таблицы с параметрами и тиражами следует заполнять только цены. Не вносите изменения в саму форму и параметры, т.к. в этом случае, система не сможет распределить цены по тиражам.**
{% endhint %}

&#x20;&#x20;

2\. создать произвольную таблицу  exel с двумя столбцами: тираж/кол-во и цена за тираж. Заполненные данные из таблицы скопируйте и вставьте в поле внизу тиражей («Скопируйте цены для тиражей….»). Цены распределятся по тиражам.

### **Таблица с ручным вводом тиража**

**Пример таблицы** для заполнения при **выборе ручного ввода** количества клиентом  (Калькуляция →Модуль Тиражность → Ручной ввод):

<figure><img src="../../../../.gitbook/assets/image (3861).png" alt=""><figcaption></figcaption></figure>

Для корректного расчета транспортных расходов под каждой позиций можно откорректировать вес готового изделия в зависимости от параметров (поле _Вес (гр.) за шт_.)

В случае изменения параметров  в калькуляции продукта, цены в табличной калькуляции следует удалить полностью (через кнопку "Удалить все"), сформировать таблицу заново и заполнить цены заново по комбинациям новых параметров. Поэтому, рекомендуем приступать к заполнению цены после занесения всех модулей и операций.

Если параметры не меняются, но требуется добавить новые тиражи (при выборе в модуле "Тиражность"→ Выбор из списка), удалять и вносить заново цены не нужно.

&#x20;Достаточно внести тиражи в модуле "Тиражность" и зайти в табличную калькуляции. Новые тиражи автоматически добавятся в таблицу.

{% hint style="info" %}
**Обратите внимание**, что цена формируется ОТ определенного кол-ва изделий, и цена соответственно указана за ШТУКУ, а не за введённое значение слева
{% endhint %}
