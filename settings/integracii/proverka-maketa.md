# Проверка макета

### Создание правила проверки

Чтобы создать правило проверки макетов в разделе Настройки→ Интеграции → Проверка макетов → щелкните мышкой на название интеграции

![](<../../.gitbook/assets/image (3335).png>)



и во вкладке Правила проверки нажмите кнопку "Добавить" в правом верхнем углу

![](<../../.gitbook/assets/image (3280).png>)



Задайте _Название_ правилу проверки макетов и _выберите версию готового PDF файла_ (пока доступна только версия 1.4).&#x20;

В поле _Quality (сжатие изображений, для фотоконструктора, %)_ вы можете установить процент сжатия фотографий. В этом случае все изображения, загруженные с проверкой будут сжаты до указанного процента. Действует только для конструкторов фотопродукции.

В поле _Погрешность на размере (мм)_ при необходимости введите возможный люфт погрешности в размере при котором макеты будут автоматически приниматься. Например, если размер продукта 90\*50 мм и введено значения погрешности 10 мм, то будут приниматься макеты размером от 80\*40 мм до 100\*60 мм.

В _Доступные форматы для загрузки_ отметьте те форматы, в которых вы планируете принимать макеты от клиентов. На форматы, не отмеченные пометкой "с проверкой" проверка действовать не будет (CDR, ZIP, RAR, DOC).

![](<../../.gitbook/assets/image (3026).png>)



После сохранения,  в поле _Продукция_  укажите продукцию, на которую будет распространяться данное правило при проверке макетов.

![](<../../.gitbook/assets/image (3482).png>)





### Настройка правил проверки

С основной вкладки перейдите на вторую вкладку "Правила проверки":

![](<../../.gitbook/assets/image (2757).png>)



Нажмите "Добавить" и выберите в выпадающем списке параметры для проверки:

![](<../../.gitbook/assets/image (3336).png>)



**1) Проверка шрифтов** **-** если в загруженном макета будут незакривленные шрифты, система автоматически осуществит их корректировку (закривление).

**2) Цветовое пространство -** система проверяет макет на цветовое пространство. По желанию может заменить одно цветовое пространство на другое.

**3) Размеры макета -** проверка макета на соответствие требуемым размерам (внесенным в Продукт → Калькуляция → Размеры). Если в проверке указано значение погрешности на размере в мм., проверка будет учитывать размер с учетом погрешности.

**4) Качество растра (dpi) -**  поиск в макете растровых изображений и проверка их на dpi. Если разрешение изображения не соответствует вашим требованиям, система может сообщить об этом в форме предупреждения. Также система позволяет назначить верхний порог разрешения. Если разрешение изображения выше порога, то качество растра понижается до него. Если разрешение изображения ниже или соответствует значению порога, то качество растра остается без изменения.

Если у выбранного параметра могут быть допустимые и недопустимые значения — укажите истинное значение в соответствующем столбце.

![](<../../.gitbook/assets/image (2219).png>)



В случае несоответствия установленному значению нужно указать варианты ответа системы:

&#x20;**• предупреждение** (с возможностью проигнорировать),&#x20;

**• критическое предупреждение** (нет возможности продолжить оформление),

&#x20;**• автоматическое исправление ошибки системой**.

Если требуется исправление какого-либо параметра — в последнем столбце указать действие, которое необходимо произвести.

![](<../../.gitbook/assets/image (2983).png>)



Можно создать отдельные правила для проверки макетов разной продукции.

![](<../../.gitbook/assets/image (513).png>)





### Отключение и удаление правила проверки макетов

Чтобы отключить правила проверки макетов воспользуйтесь переключателями напротив каждого правила

![](<../../.gitbook/assets/image (2979).png>)



Чтобы удалить правило нажмите на кнопку удалить напротив правила.

![](<../../.gitbook/assets/image (3294).png>)
