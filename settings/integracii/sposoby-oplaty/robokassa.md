# ROBOKASSA (РОБОКАССА)

### Шаг 1. Регистрация на сайте поставщика услуг

Для подключения сервиса ROBOKASSA (РОБОКАССА) вам необходимо предварительно зарегистрироваться на сайте [https://robokassa.com/](https://robokassa.com/)

{% hint style="info" %}
**Важно!** ROBOKASSA предъявляет следующие требования к контенту сайта:

* наличие реквизитов организации на сайте
* наличие раздела с платными товарами и услугами, за которые будет взиматься плата
{% endhint %}





### Шаг 2. Настройки личного кабинета ROBOKASSA

После прохождения процедуры регистрации (заключения договора) и проверки данных организации, в Личном Кабинете  ROBOKASSA зайдите в Меню → Управление → Мои магазины →  Технические настройки → заполните необходимые параметры:

* **Идентификатор магазина** — адрес вашего сайта, придуманный Вами при создании магазина. Это название будет видеть клиент при совершении оплаты в интерфейсе ROBOKASSA.
* **Алгоритм расчета хэша** — оставляем вариант MD5
* **Пароль 1 и 2** — необходимо сгенерировать, нажав на кнопку "Сгенерировать"
* **Result url** — **данный url вы можете взять в админ-панели сайта → Настройки → Интеграции → Способы оплаты →  ROBOKASSA → внизу страницы**
* **Метод отсылки данных** — по всем url выбрать GET
* **Success url** — в этом поле уже имеется адрес вашего сайта. Необходимо после слэша (/) прописать "success" (пример, https://demo.wow2print.com/success). Это страница, на которую будет перенаправлен покупатель, после успешного платежа.
* **Fail url** — также прописать в названии сайта после слэша "fail" (пример https://demo.wow2print.com/fail). Это страница, на которую будет перенаправлен покупатель после не успешного платежа, отказа от оплаты.

### Шаг 3. Настройка интеграции ROBOKASSA в админ-панели Сайта

На сайте в Админ-панели → Настройки → Интеграции → Способы оплаты, щелкните мышкой на ROBOKASSA

![](<../../../.gitbook/assets/image (1511).png>)

Заполните поля в открывшейся форме:

* В поле **Компания-продавец** выберите из списка нужную компанию. В списке отображаются все компании, данные о которых заполнены в Настройки → Оплата → Реквизиты
* Отметьте переключателями для кого будет доступна оплата с помощью данной интеграции.&#x20;

{% hint style="info" %}
- ROBOKASSA предназначена как способ оплаты только для клиентов физических лиц. Для юридических лиц предусмотрен вариант Оплата по счету.
{% endhint %}

* в поля **Идентификатор магазина, Пароль 1, Пароль 2** вставьте данные из Технических настроек личного кабинета ROBOKASSA&#x20;
* в пункте **Язык шлюза** укажите язык
* выберите как передавать товар в платежную систему: **как товар** или **как услугу.** Вы можете выбрать вариант передачи информации о продаже в качестве оказанной услуги (по умолчанию указывается товар). Название товара (услуги), передаваемого в платежную систему, формируется из параметров «Наименование для счёта» в продукции, или «Наименование для счёта» в конкретном товаре в карточке заказа. Если поле «Наименование для счёта» не заполнено, будет передаваться наименование продукта
* **Режим оплаты**: выберите Тестовый или Рабочий режим.

При первой настройке ROBOKASSA необходимо проверить работу интеграции. При помощи Тестового режима проверяется работоспособность самой интеграции (правильность заполнения ключей и url адресов). Если оплата проходит и вас перенаправляет на страницу успешной оплаты, значит интеграция настроена корректно и можно переключить на рабочий режим.

Подробнее о проверке в тестовом режиме [**https://docs.robokassa.ru/#2387**](https://docs.robokassa.ru/#2387)



{% hint style="info" %}
При оплате клиентом на сайте через сервис ROBOKASSA статус Оплаты заказа проставляется автоматически. За смену статуса оплаты отвечает ссылка, которая указана внутри самой интеграции (в самом низу страницы). Именно на нее делается запрос из банка, чтобы сменился статус Оплаты.&#x20;

Если статус оплаты не сменяется, проверьте наличие этой ссылки в настройках Личного Кабинета вашего эквайринга (данные поля могут иметь названия: Result URL, Service URL и подобные)
{% endhint %}
