# Заказы/Товары/Услуги

## **Список заказов**

### **Общее представление**

> Магазин → Заказы

![](<../../.gitbook/assets/image (2026).png>)



### **Работа со списком заказов**

Заказы фильтруются по:

* менеджерам;
* статусам заказа;
* его типу (услуга/продукт);
* статусу оплаты/гарантии оплаты;
* способу оплаты.

Заказы сортируются по:

* номеру;
* клиенту;
* доставке;
* дате заказа.



### Настройки списка заказов

Вы можете настроить отображение заказов по разным параметрам, нажав на кнопку <img src="../../.gitbook/assets/Screenshot_1 (10).png" alt="" data-size="original">&#x20;

![](<../../.gitbook/assets/image (1645).png>)



Отметьте в открывшемся окне нужные столбцы для отображения.

![](<../../.gitbook/assets/image (1062).png>)



Доступные параметры:

* [x] номер;
* [x] клиент;
* [x] стоимость;
* [x] менеджер;
* [x] статус/гарантия оплаты (ГО);
* [x] тип;
* [x] плательщик;
* [x] доставка;
* [x] дата заказа;
* [x] способ оплаты;
* [x] тип оплаты.



## **Группировка по вкладкам**

### Общее описание

Все заказы группируются по вкладкам в зависимости от установленной связи статусов заказов и вкладок.

Вкладки и статусы создаются  в

> Настройки → Статусы → Статусы заказов



Назначенные статусы сопоставляются с вкладками, при изменении статуса заказа, он переходит в сопоставленную со статусом вкладку.



### Вкладка Неоплаченные

Здесь же, в разделе [Статусы заказов](../../settings/statusy/statusy-zakazov/), проставляется [срок оплаты заказа](../../settings/statusy/statusy-zakazov/srok-oplaty-zakaza.md) в часах:

![](<../../.gitbook/assets/image (3160).png>)



По истечению указанного времени заказ автоматически перемещается во вкладку Неоплаченные:

![](<../../.gitbook/assets/image (1048).png>)



{% hint style="info" %}
Если по заказу, который находится во вкладке Неоплаченные пройдет оплата от клиента, то он автоматически переместится во вкладку Новые.
{% endhint %}



### Вкладка Неразобранные

Заказы от новых, ранее не зарегистрированных, клиентов по умолчанию попадают в вкладку Неразобранные.&#x20;

После нажатия "Принять", клиент закрепляется за менеджером, принявшим заказ.

![](<../../.gitbook/assets/image (1119).png>)



В случае, если вы используете [автоматическое распределение заказов](../../settings/polzovateli-1/nastroika-menedzherov.md) по менеджерам, данная вкладка будет отсутствовать.&#x20;

## Изменение статуса

В нижней части раздела находится панель управления статусами и движением по вкладкам.

![](<../../.gitbook/assets/image (1790).png>)

Для изменения статуса заказа/заказов нужно отметить галочкой их в чек-боксе слева (квадратик). В появившемся всплывающем окне выберите статус и нажмите "Изменить". У заказа поменяется статус и он переместится во вкладку, сопоставленную с выбранным статусом. Настройки сопоставления:

> Настройки → Статусы → [Статусы заказов](../../settings/statusy/statusy-zakazov/)



## Распределение заказов по менеджерам

### Варианты

Система позволяет настроить распределение заказов от новых клиентов:

1. Автоматическое, в порядке очереди между менеджерами (пользователями);
2. По принципу "Кто успел — тот и съел", то есть каждый менеджер принимает в работу столько заказов от новых клиентов, сколько доступно во вкладке Неразобранные.&#x20;

Уведомление о поступлении нового заказа будет приходить:

* на почту менеджера, ответственного за заказ и закрепленного за конкретным клиентом (в случае Автоматического распределения заказов - **всегда** на почту менеджера)
* на почту типографии, указанную в [Контактных данных](../../settings/drugie-nastroiki/kontaktnye-dannye.md), в случае ручного распределения заказов. Если заказ от нового клиента, за которым **не** закреплен менеджер, необходимо принять заказ (кнопка "Принять").

{% hint style="info" %}
1. Менеджер — это пользователь, имеющий доступ "Возможность принимать заказ" в разделе Настройки → Пользователи → [Роли и доступы](../../settings/polzovateli-1/roli-i-dostupy.md);<br>
2. Вкладка Неразобранные отображается только в том случае, когда у вас НЕ установлен параметр распределения заказов — "Распределять заказы автоматически";<br>
3. После назначения заказа какому-либо менеджеру, новый клиент закрепляется за ним, и последующие его заказы также будут назначены этому менеджеру автоматически.
{% endhint %}



### Настройка вариантов

Настроить данную функцию можно в:

> Настройки → Пользователи → Настройка менеджеров → Распределять заказы автоматически (выбрать) → Сохранить

## Удаление заказа

Чтобы удалить заказ из списка,  выберите его и нажмите кнопку "Удалить" в правом нижнем углу.

![](<../../.gitbook/assets/image (2483).png>)

{% hint style="info" %}
При удалении заказа он попадет во вкладку Помеченные на удаление.
{% endhint %}

![](<../../.gitbook/assets/image (2566).png>)



{% hint style="info" %}
При удалении заказа удаляются и закрывающие документы на заказ.
{% endhint %}

## **Экспорт заказов**

Информацию о заказах по выбранным параметрам можно скачать в excel-файл, нажав на кнопку "Скачать в файл".&#x20;



![](<../../.gitbook/assets/image (3573).png>)



{% hint style="info" %}
**Внимание!** Данные по заказам сохраняются в таблицу строго по параметрам, которые были выбраны в настройках параметров отображения (в [Столбцах отображения](./#nastroiki-spiska-zakazov)).
{% endhint %}
